From: Fabian Huch <huch@in.tum.de>

Content of the AFP website (well, excluding the submission system) is
fully static and as a result serving requests is very efficient --
hence, we should not have a problem with AI scrapers here.

In this case the server was unresponsive but the logs do not indicate
any concrete problem. If the problem re-appears I will probably migrate
the AFP to Caddy [1], which is much simpler to configure and has proven
to be very stable so far.

Fabian

[1]: https://caddyserver.com/
