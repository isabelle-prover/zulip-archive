Hi Pedro,

I see the same thing.

The file uses UTF-8, but for some reason (there is no proper HTML
version nor encoding specified when you inspect the document or display
the source of the page), it is displayed as ISO-8859-1.
Since this is a pure text file, I guess the headers are generated by the
http server or the file is displayed according to the default settings
of the browser.
It seems that before HTML 5, the default encoding has been ASCII then
ISO-8859-1. Starting with HTML 5, it is UTF-8, see
http://www-db.deis.unibo.it/courses/TW/DOCS/w3schools/charsets/default.asp.html

I guess that because there is no html version specified in the headers
for this file, the browser conservatively uses ISO-8859-1.

Note that I may be completely wrong, I am not a specialist of web
servers and clients...

In Firefox, in the "Display" menu, there is a "Fix text encoding" item
that really fixes the issue (I am not sure about the name of the menu
and item because they are displayed in French on my system).
