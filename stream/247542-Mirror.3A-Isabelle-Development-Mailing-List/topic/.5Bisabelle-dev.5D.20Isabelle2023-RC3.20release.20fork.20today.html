The tag Isabelle2023-RC3 and post-release point f467ff4aa8f9 are already
published on isabelle-dev. From now on, all changes are for post-Isabelle2023
development.

While afp-devel is still on pre-Isabelle2023, we need to take care that
nothing breaks over there. As of AFP/4605c928f00e, everything works with
Isabelle2023-RC3 (tested on arm64_32-darwin).

Today, I spent several hours longer than anticipated, and did not fully manage
to get everything into RC3 that was planned:

* The cluster build tools are hardly changed wrt. RC2: several fine points
still need to be sorted out to make this work properly.

* The bundled VSCodium is still at 1.70.1, as in Isabelle2022. I have
invested 1-2 days trying to build current 1.81.0.23216, but failed. It works
on Linux and presumably macOS, but Windows as cross-build on Linux failes due
to oddities with rcedit.exe via wine.

There is no particular reason to update VSCodium, apart from noisy crowds from
the "latest-is-greatest" cult. And since there is a new release every month,
any current version will be outdated quickly.
