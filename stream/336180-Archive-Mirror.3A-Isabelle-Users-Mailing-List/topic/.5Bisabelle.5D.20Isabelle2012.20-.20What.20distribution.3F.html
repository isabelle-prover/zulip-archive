Our Isabelle2012 standard distribution bundle attempts to be
all-inclusive and as self-contained as feasible, but refers to standard
C/C++ libraries from the host system. Here we are essentially gambling
to have it work by accident for the majority of users: binaries are
compiled on some Linux system that is neither too new nor too old.

For Isabelle2012 that was Ubuntu 10.04 LTS, from April 2010. According to
wikipedia, CentOS 5.4 from 2009, so it is plausible that it provides an
older libc that does not work with our binaries.

See also
http://isabelle.in.tum.de/repos/isabelle/file/Isabelle2012/Admin/PLATFORMS
for the internal explanations how Isabelle distributions are made as
platform-independent as possible.

Of course, you could try to recompile binaries that we ship, but that will
take a lot of time to reconstruct. It is easier to install a more recent
Linux, such as Ubuntu 12.04.1 or anything else in the 2011-2012 range.
