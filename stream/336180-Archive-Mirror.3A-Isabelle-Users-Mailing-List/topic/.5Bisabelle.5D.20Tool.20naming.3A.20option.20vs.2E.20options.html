From: Florian Haftmann <florian.haftmann@informatik.tu-muenchen.de>
I found the following naming inconsistency in Isabelle2017-RC2:

  • There is the tool »isabelle option«…

  • …but it advertizes itself as »isabelle options«:

Isabelle2017-RC2/bin/isabelle option

Usage: isabelle options [OPTIONS] [MORE_OPTIONS ...]

Options are:
-b include $ISABELLE_BUILD_OPTIONS
-g OPTION get value of OPTION
-l list options
-x FILE export options to FILE in YXML format

Report Isabelle system options, augmented by MORE_OPTIONS given as
arguments NAME=VAL or NAME.

  • The Isabelle System Manual also speaks about »isabelle options«
