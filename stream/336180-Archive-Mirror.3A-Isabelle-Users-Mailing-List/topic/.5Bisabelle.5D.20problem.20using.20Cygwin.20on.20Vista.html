This looks bad -- basic Poly/ML fails on your machine. These precompiled
binaries have been created on XP/SP3, so there is some chance that there is a
problem running them on Vista, although I've heard from people who managed
that.

Since this is Poly/ML 5.2 from the Isabelle2008 distribution, I do recommend to
try PolyML/ML 5.2.1 from the current Isabelle2009 distribution.

Also make sure that your Cygwin is up-to-date.

If Poly/ML 5.2.1 still crashes on you, just compile it yourself from the
sources, according to the included README.

Alternatively, you can also work with a virtual Linux installation, say Ubuntu
on top of Virtualbox (it usually takes less than 1h to install the whole
system).
